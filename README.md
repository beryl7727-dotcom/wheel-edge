# Wheel Edge Trading Dashboard

A local-first, decision-support trading dashboard for managing wheel strategies (TSLA, IBIT, BBAI, XLF, Oil ETFs).

## 🎯 Current Phase: Shell (Phase 1)

This is the **UI foundation only**. All pages are built and fully routable. Business logic (calculators, integrations, database operations) comes in later phases.

### What's Included
- ✅ Complete routing with React Router
- ✅ Persistent sidebar navigation
- ✅ 8 fully functional page shells
- ✅ Design system (Playfair Display + Inter, gradients, theme)
- ✅ Mock data for all sections
- ✅ Account snapshot widget
- ✅ Zustand state management setup
- ✅ Responsive layout (desktop-first)
- ✅ **Electron setup for desktop app** (.exe, .dmg, .AppImage)

### What's Not Yet Built
- ❌ IndexedDB integration
- ❌ CSV import/export
- ❌ Calculator logic (Greeks, scenarios, rolls)
- ❌ Calendar features (week/agenda views, events)
- ❌ Editable tables
- ❌ Real-time data feeds
- ❌ Mobile responsiveness

## 🖥️ Desktop App (Electron)

### Why Electron?
- Single codebase runs on Windows, macOS, and Linux
- No web server needed
- Works offline
- Native desktop integration (system tray, file dialogs, etc.)
- Easy distribution via installers

### Development (Hot Reload)

```bash
npm run electron-dev
```

This:
1. Starts React dev server on port 3000
2. Waits for it to be ready
3. Launches Electron window pointing to localhost:3000
4. Hot-reloads on code changes
5. Opens DevTools automatically

**Exit:** Close the Electron window or press Ctrl+C in terminal

### Production Build

```bash
npm run electron-build
```

Creates optimized executable for your current OS:
- **Windows:** `dist/Wheel Edge.exe` (NSIS installer + portable)
- **macOS:** `dist/Wheel Edge.dmg` (DMG installer)
- **Linux:** `dist/Wheel Edge.AppImage` (AppImage executable)

**Output location:** `dist/` folder

### Cross-Platform Build

Build for all platforms (requires build tools for each):

```bash
npm run electron-build-all
```

**Requirements:**
- Windows: Visual Studio Build Tools (for NSIS)
- macOS: Xcode Command Line Tools
- Linux: build-essential

### Distribution

Once built, distribute:

**Windows Users:**
- Share `.exe` file → they run installer
- Or share portable `.exe` → no install needed

**macOS Users:**
- Share `.dmg` → drag to Applications folder

**Linux Users:**
- Share `.AppImage` → make executable, run

## 🎨 Design System

### Colors
- **Primary Gradient:** Purple (#a855f7) → Blue (#3b82f6)
- **Success Gradient:** Green (#10b981) → Emerald (#059669)
- **Neutral:** Slate 50–900
- **Accents:** Red, orange, yellow (for positions, alerts)

### Typography
- **Headers:** Playfair Display (serif), bold, 700–900 weight
- **Body:** Inter (sans-serif), 400–600 weight
- **Sizes:** Maintain 4px baseline grid

### Components
- Rounded corners: 12px (0.75rem) default
- Shadows: soft (2px 8px), medium (4px 16px)
- Spacing: 4px, 8px, 16px, 24px, 32px, 40px
- Buttons: Gradient backgrounds, hover shadow, active scale-down

### Responsive Behavior
- **Desktop:** Sidebar 256px fixed, main content flex-1
- **Tablet:** [Future] Collapsible sidebar
- **Mobile:** [Future] Hamburger menu, single column

## 💡 Development Notes

### Adding New Pages

1. Create component function in `wheel-edge-dashboard.jsx`
2. Add Route in `App` component
3. Wrap in `<LayoutWrapper>` to get sidebar
4. Follow design patterns from existing pages

Example:
```javascript
function NewPage() {
  const data = useWheelStore((state) => state.collection);
  
  return (
    <LayoutWrapper>
      <div className="p-8 space-y-6">
        {/* Your content */}
      </div>
    </LayoutWrapper>
  );
}
```

### Updating Mock Data

All mock data is defined at the top of `wheel-edge-dashboard.jsx`:
- `MOCK_POSITIONS`
- `MOCK_JOURNAL`
- `MOCK_CALENDAR`
- `MOCK_WATCHLIST`
- `MOCK_WHEEL_CYCLES`
- `MOCK_SCENARIOS`
- `MOCK_INCOME`
- `ACCOUNT_SNAPSHOT`

Modify as needed for testing.

### Using Zustand Store

Get state:
```javascript
const positions = useWheelStore((state) => state.positions);
```

Update state:
```javascript
const updatePosition = useWheelStore((state) => state.updatePosition);
updatePosition(1, { premium: 500 });
```

### Styling

All styling uses Tailwind classes. No CSS-in-JS or external CSS files needed (except global index.css).

Common patterns:
```javascript
// Gradient backgrounds
className="bg-gradient-to-r from-purple-500 to-blue-500"

// Conditional styles
className={`px-4 py-2 ${isActive ? 'bg-purple-500 text-white' : 'bg-slate-200'}`}

// Nested spacing
className="space-y-6" // 24px gap between children
className="gap-4"     // 16px gap in flex/grid
```
